  Key Findings: Building a Plexus-RPC Microservice (Activation)

  What You Need to Know

  Activations are self-contained microservices in the Plexus ecosystem that expose typed RPC methods. They're defined in the substrate codebase (not in hub-codegen, which is just the client generator).

  The Pattern for Building a New Activation

  1. Define in Substrate using the #[hub_method] macro:
  // In substrate codebase
  #[derive(Serialize, Deserialize, JsonSchema)]
  #[serde(tag = "type", rename_all = "snake_case")]
  pub enum AgentEvent {
      Started { agent_id: String },
      Progress { message: String },
      Result { output: String },
      Complete,
  }

  impl AgentOrchestratorActivation {
      #[hub_method(streaming, description = "Orchestrate Claude Code agents")]
      async fn orchestrate(&self, params: OrchestrateParams)
          -> impl Stream<Item = AgentEvent> {
          // Your implementation calling Claude Code
      }
  }

  2. Key Requirements:
    - Use internally-tagged enums (#[serde(tag = "type")])
    - All methods return streams (even single-value returns)
    - Use snake_case for Rust types (auto-converts to camelCase in TS)
    - Methods get wrapped in PlexusStreamItem envelope automatically
  3. Communication Protocol:
    - Transport: WebSocket + JSON-RPC 2.0
    - Envelope: All responses wrapped in PlexusStreamItem (data/progress/error/done)
    - Namespace: Hierarchical (e.g., cone, arbor, or claude_orchestrator)

  Architecture

  Your New Activation (in substrate)
      ↓ exposes JSON Schema via #[hub_method]
  Synapse (introspects schemas)
      ↓ generates IR (Intermediate Representation)
  hub-codegen (this repo)
      ↓ generates TypeScript/Rust clients
  Type-Safe Clients (can call your activation)

  Relevant Files for Reference

  - Transport layer generation: src/generator/typescript/transport.rs:104-392
  - Method schema spec: docs/architecture/16679174944041108735_method-schema-spec.md
  - Multi-hub protocol: docs/architecture/16679517135570018559_multi-hub-transport-envelope.md
  - Rust client examples: docs/architecture/16677426370165267711_rust_examples.md

  Next Steps for Your Claude Code Orchestrator

  You'll need to:
  1. Create the activation in the substrate codebase (not here)
  2. Define methods like orchestrate(), spawn_agent(), etc.
  3. Implement streaming event types for orchestration progress
  4. Call into Claude Code API from within your activation
  5. Generate clients using this hub-codegen tool

  The activation would live alongside existing ones like cone (LLM agents) and arbor (conversation trees) in the substrate codebase.
