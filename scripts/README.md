# Hub Codegen Pipeline Scripts

## update-rust-client.sh

Automated pipeline for generating and deploying Rust client from Plexus IR.

### What it does

1. 🔄 **Auto-generates IR** (optional) - Calls synapse to fetch latest IR from substrate
2. ✅ **Validates** target repository is git-clean (no uncommitted changes)
3. 🔨 **Generates** new Rust client code to temporary directory
4. ✔️ **Verifies** generated code compiles successfully
5. 📦 **Replaces** target repository contents (preserves .git, README.md, .gitignore)
6. 🏷️ **Versions** using timestamp + IR hash (e.g., `0.1.20260125041323-32e46854`)
7. 💾 **Commits** with detailed message including IR hash and timestamp
8. 🏷️ **Tags** with version number
9. 📋 **Reports** next steps for pushing

### Usage

**Mode 1: Use existing IR file**

```bash
# 1. Generate IR manually
synapse -P 44410 substrate -i > /tmp/substrate-ir.json

# 2. Run pipeline
./scripts/update-rust-client.sh /tmp/substrate-ir.json /path/to/plexus-rust-client

# 3. Push changes (if satisfied)
cd /path/to/plexus-rust-client
git push origin master
git push origin <tag-name>
```

**Mode 2: Auto-generate IR (recommended)**

```bash
# Run pipeline - it will call synapse automatically
./scripts/update-rust-client.sh --generate /path/to/plexus-rust-client

# With custom connection settings
./scripts/update-rust-client.sh --generate /path/to/plexus-rust-client \
  --host 127.0.0.1 \
  --port 44410 \
  --backend substrate

# Push changes (if satisfied)
cd /path/to/plexus-rust-client
git push origin master
git push origin <tag-name>
```

### Version Scheme

Format: `0.1.<timestamp>-<ir-hash-8-chars>`

- `0.1` = major.minor (static for now)
- `<timestamp>` = YYYYMMDDHHmmss (UTC)
- `<ir-hash-8-chars>` = First 8 chars of IR hash for traceability

Example: `0.1.20260125041323-32e46854`

### Safety Features

- 🔍 **Validates synapse connectivity** (auto-generate mode) - Checks substrate is reachable
- ✅ **Validates IR is valid JSON** - Won't proceed with corrupted IR
- ❌ **Fails if target has uncommitted changes** - Won't clobber your work
- ❌ **Fails if generated code doesn't compile** - Won't deploy broken code
- ✅ **Generates to temp directory first** - Original repo untouched until verified
- ✅ **Preserves README.md and .gitignore** - Keeps repository documentation
- 🧹 **Auto-cleanup** - Temporary IR files cleaned up automatically

### Quick Reference

**Auto-generate mode (simplest):**
```bash
./scripts/update-rust-client.sh --generate <repo-dir>
```

**Custom connection:**
```bash
./scripts/update-rust-client.sh --generate <repo-dir> --port 4444 --backend substrate
```

**Manual IR mode:**
```bash
./scripts/update-rust-client.sh <ir-file.json> <repo-dir>
```

**Connection options:**
- `--host HOST` - Substrate host (default: 127.0.0.1)
- `--port PORT` - Substrate port (default: 44410)
- `--backend NAME` - Backend name (default: substrate)

### Automation

**Simple automation (recommended):**

```bash
#!/bin/bash
# Auto-update and push script

TARGET_REPO="/path/to/plexus-rust-client"
SUBSTRATE_HOST="127.0.0.1"
SUBSTRATE_PORT="44410"

# Run pipeline (auto-generates IR)
./scripts/update-rust-client.sh --generate "$TARGET_REPO" \
  --host "$SUBSTRATE_HOST" \
  --port "$SUBSTRATE_PORT"

# Auto-push (use with caution!)
cd "$TARGET_REPO"
git push origin master
git push --tags
```

**With manual IR (if you want to inspect it first):**

```bash
#!/bin/bash
# Auto-update and push script

IR_FILE="/tmp/substrate-ir.json"
TARGET_REPO="/path/to/plexus-rust-client"

# Generate IR
synapse -P 44410 substrate -i > "$IR_FILE"

# Inspect IR if needed
jq '.irHash' "$IR_FILE"

# Run pipeline
./scripts/update-rust-client.sh "$IR_FILE" "$TARGET_REPO"

# Auto-push (use with caution!)
cd "$TARGET_REPO"
git push origin master
git push --tags
```

### Troubleshooting

**Error: "Target repository has uncommitted changes"**
- Commit or stash changes in target repository first
- Run `git status` in target to check

**Error: "Generated code does not compile"**
- Check hub-codegen output for syntax errors
- Verify IR is valid (run `jq . < ir-file.json`)
- Report bug with IR file

**Error: "hub-codegen binary not found"**
- Build hub-codegen first: `cargo build --release --features rust`
