# Rust Code Generator Architecture

**Status:** Implemented
**Date:** 2025-01-24
**Author:** Claude Sonnet 4.5

## Overview

The Rust code generator creates a strongly-typed, async Rust client from Synapse IR. It generates a modular, namespace-scoped codebase that compiles to a production-ready WebSocket client library.

## Architecture

### High-Level Design

```
Synapse IR (JSON)
    ↓
hub-codegen (Rust generator)
    ↓
Generated Rust Client
    ├── Core transport (PlexusStreamItem, WebSocket)
    ├── Namespace modules (types + methods per namespace)
    └── Base client (connection management)
```

### Key Principles

1. **Namespace Modularization**: Each Plexus namespace (e.g., `cone`, `arbor`, `bash`) gets its own module file containing both types and methods
2. **Cross-Namespace Imports**: Automatic detection and import of types used across namespaces
3. **Type Safety**: All PlexusStreamItem unwrapping handled automatically, returning domain types
4. **Streaming Support**: Proper async Stream types for streaming methods

## Module Structure

### Generated Directory Layout

```
plexus-client/
├── Cargo.toml
└── src/
    ├── lib.rs              # Module declarations, re-exports
    ├── types.rs            # Core transport types (PlexusStreamItem)
    ├── client.rs           # Base PlexusClient with WebSocket
    ├── cone.rs             # Cone namespace (types + methods)
    ├── arbor.rs            # Arbor namespace (types + methods)
    ├── bash.rs             # Bash namespace (types + methods)
    └── ...                 # Other namespace modules
```

### File Organization

**`lib.rs`**:
- Module declarations for all namespaces
- Re-exports PlexusClient
- Re-exports PlexusStreamItem

**`types.rs`**:
- PlexusStreamItem enum (protocol envelope)
- StreamMetadata struct
- Core transport types only

**`client.rs`**:
- PlexusClient struct (holds WebSocket URL)
- `call_stream()` - returns `Pin<Box<dyn Stream<Item = Result<PlexusStreamItem>>>>`
- `call_single()` - unwraps stream, returns first Data event

**Namespace modules** (e.g., `cone.rs`):
- Types defined in that namespace
- Methods for that namespace
- Cross-namespace imports at top
- Standalone async functions (not impl block)

## Code Generation Flow

### 1. Entry Point (`mod.rs::generate()`)

```rust
pub fn generate(ir: &IR) -> Result<GenerationResult>
```

**Steps:**
1. Validate IR version (requires 2.0)
2. Collect warnings for unknown types
3. Generate `lib.rs` (module declarations)
4. Generate `types.rs` (core transport only)
5. Generate `client.rs` (base client)
6. Generate namespace modules via `client::generate_namespace_modules()`
7. Generate `Cargo.toml`
8. Return `GenerationResult { files, warnings }`

### 2. Type Generation (`types.rs`)

**Core Types** (`generate_core_types()`):
```rust
pub enum PlexusStreamItem {
    Data { metadata: StreamMetadata, content_type: String, content: serde_json::Value },
    Progress { metadata: StreamMetadata, message: String, percentage: Option<f64> },
    Error { metadata: StreamMetadata, message: String, code: Option<String> },
    Done { metadata: StreamMetadata },
}
```

**Namespace Types** (`generate_types_for_namespace()`):
- Filter types by `td_namespace`
- Sort for deterministic output
- Generate each typedef (struct/enum/alias/primitive)

### 3. Client Generation (`client.rs`)

**Base Client**:
```rust
pub struct PlexusClient {
    url: String,
}

impl PlexusClient {
    pub fn new(url: impl Into<String>) -> Self;
    pub(crate) async fn call_stream(...) -> Result<Pin<Box<dyn Stream<...>>>>;
    pub(crate) async fn call_single<T>(...) -> Result<T>;
}
```

**Namespace Modules** (`generate_namespace_modules()`):

For each namespace:
1. Collect methods in namespace
2. Collect types in namespace
3. Detect cross-namespace type references
4. Generate imports
5. Generate types
6. Generate methods

**Method Signature**:
```rust
// Standalone function (not impl method)
pub async fn chat(
    client: &PlexusClient,
    ephemeral: Option<bool>,
    identifier: ConeIdentifier,
    prompt: String,
) -> Result<ChatEvent>
```

### 4. Streaming vs Non-Streaming

**Non-streaming**:
```rust
pub async fn check(client: &PlexusClient) -> Result<HealthEvent> {
    client.call_single("health.check", serde_json::Value::Null).await
}
```

**Streaming**:
```rust
pub async fn execute(client: &PlexusClient, command: String)
    -> Result<Pin<Box<dyn Stream<Item = Result<BashEvent>> + Send>>>
{
    let stream = client.call_stream("bash.execute", json!({ "command": command })).await?;

    // Filter and transform PlexusStreamItem -> BashEvent
    let typed_stream = stream.filter_map(|item| async move {
        match item {
            Ok(PlexusStreamItem::Data { content, .. }) => {
                match serde_json::from_value::<BashEvent>(content) {
                    Ok(data) => Some(Ok(data)),
                    Err(e) => Some(Err(e.into())),
                }
            }
            Ok(PlexusStreamItem::Error { message, code, .. }) => {
                Some(Err(anyhow!("Plexus error{}: {}", ...)))
            }
            Ok(PlexusStreamItem::Progress { .. }) => None, // Skip
            Ok(PlexusStreamItem::Done { .. }) => None, // End
            Err(e) => Some(Err(e)),
        }
    });

    Ok(Box::pin(typed_stream))
}
```

## Key Implementation Details

### Rust Keyword Escaping

Fields/parameters that conflict with Rust keywords are escaped with `r#`:

```rust
pub struct PackageConfig {
    pub r#type: PackageType,  // 'type' is a Rust keyword
    pub publish: bool,
}
```

**Keywords escaped:** `type`, `async`, `match`, `loop`, `self`, `mod`, `use`, etc.

**Implementation**:
```rust
fn escape_keyword(name: &str) -> String {
    match name {
        "as" | "async" | "await" | "break" | "const" | ... => format!("r#{}", name),
        _ => name.to_string(),
    }
}
```

### Cross-Namespace Type Imports

When types from one namespace are used in another:

```rust
// In arbor.rs - uses UUID from cone namespace
use crate::cone::UUID;

pub struct TreeQuery {
    pub tree_id: UUID,
}
```

**Algorithm** (`collect_cross_namespace_type_imports()`):
1. Scan method parameters and return types
2. Scan struct fields
3. Scan enum variant fields
4. Recursively follow `RefArray`, `RefOptional` type refs
5. For each `RefNamed`, check if it's from a different namespace
6. Generate imports grouped by source namespace

### Missing Type Handling

When IR references non-existent types, fall back to `serde_json::Value`:

```rust
fn type_ref_to_rust(tr: &TypeRef, ir: &IR) -> String {
    match tr {
        TypeRef::RefNamed(qn) => {
            let full_name = qn.full_name();
            if ir.ir_types.contains_key(&full_name) {
                to_pascal(&qn.local_name())
            } else {
                "serde_json::Value".to_string()  // Type doesn't exist
            }
        }
        // ...
    }
}
```

### Multi-Line Documentation

IR descriptions with embedded newlines are split and formatted:

```rust
let doc_comment = if let Some(desc) = &typedef.td_description {
    desc.lines()
        .map(|line| format!("/// {}\n", line.trim()))
        .collect::<String>()
} else {
    String::new()
};
```

## Problems Solved

### 1. Duplicate Type Definitions

**Problem**: Types like `ChatEvent`, `UUID` defined in multiple namespaces (claudecode, cone).

**Solution**: Namespace-scoped type generation. Each namespace's types go in its own module:
- `cone::ChatEvent`
- `claudecode::ChatEvent`

### 2. Duplicate Method Names

**Problem**: 737 compilation errors from duplicate method names across namespaces (300x `info`, 253x `schema`).

**Solution**:
- Standalone functions in namespace modules instead of single `impl PlexusClient` block
- Methods are called as `plexus_client::cone::info()` vs `plexus_client::health::info()`

### 3. Enum Variant Field Visibility

**Problem**: `pub` on enum variant fields is illegal in Rust.

**Solution**: Removed `pub` from variant fields (they inherit enum visibility).

### 4. Cross-Namespace References

**Problem**: `arbor` module uses `UUID` from `cone` namespace.

**Solution**: Automatic import generation by scanning:
- Method parameters/returns
- Struct fields
- Enum variant fields

## Usage

### Generation

```bash
cd /Users/user/dev/controlflow/hypermemetic/hub-codegen

# Build with Rust feature
cargo build --release --features rust

# Generate IR
synapse plexus -i > /tmp/plexus-ir.json

# Generate Rust client
./target/release/hub-codegen /tmp/plexus-ir.json -o /tmp/plexus-rust-client -t rust

# Verify compilation
cd /tmp/plexus-rust-client && cargo check
```

### Client Usage

```rust
use plexus_client::PlexusClient;
use futures::StreamExt;

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let client = PlexusClient::new("ws://localhost:4444");

    // Non-streaming
    let health = plexus_client::health::check(&client).await?;

    // Streaming
    let mut stream = plexus_client::bash::execute(&client, "ls".to_string()).await?;
    while let Some(event) = stream.next().await {
        println!("{:?}", event?);
    }

    Ok(())
}
```

## Comparison with TypeScript Generator

### Similarities

- Namespace modularization (separate files per namespace)
- Cross-namespace type imports
- Core transport types separate from domain types
- Streaming vs non-streaming method handling

### Differences

| Aspect | TypeScript | Rust |
|--------|-----------|------|
| Imports | Relative paths (`import { UUID } from '../cone/types'`) | Trait imports (`use crate::cone::UUID`) |
| Streaming | `AsyncIterableIterator<T>` | `Pin<Box<dyn Stream<Item = Result<T>>>>` |
| Errors | Exceptions | `Result<T, Error>` |
| Keyword conflicts | None (JS has few keywords) | `r#` prefix for keywords |
| Compilation | Runtime type checking | Compile-time type checking |
